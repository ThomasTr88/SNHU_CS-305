For this journal entry, I selected my Artemis Financial Practices for Secure Software Report as my artifact. Artemis Financial is a financial services company that needed to improve the security of its RESTful web application. The main issue the client wanted to address was the presence of security vulnerabilities in their software, including risks related to outdated dependencies, insecure communication, and improper handling of user input. Their goal was to ensure that sensitive financial data could be transmitted and processed securely while maintaining a reliable and functional application.

What I believed that I did well during the vulnerability assessment process was identifying security risks through both manual review and automated tools. I used dependency-check scans to find known vulnerabilities in third-party libraries and analyzed the results to determine which issues were real and which were false positives. Coding securely is important because it helps prevent attacks such as injection, data breaches, and unauthorized access. Software security adds value to a company by protecting customer data, maintaining trust, and reducing the risk of financial and reputational damage.

One of the more challenging parts of the vulnerability assessment was interpreting the results from the dependency-check tool. At first, it was difficult to understand which vulnerabilities were critical and which ones were false positives. However, this process was also helpful because it taught me how to research vulnerabilities using sources like the National Vulnerability Database (NVD) and how to apply suppression techniques to improve scan accuracy.

To increase layers of security, I implemented HTTPS using SSL/TLS, applied secure hashing with SHA-256, and addressed dependency vulnerabilities by updating or suppressing them when appropriate. In the future, I would continue to use tools like OWASP Dependency-Check, along with static code analysis and manual review, to assess vulnerabilities and choose the best mitigation strategies. Combining multiple tools and techniques helps create a more complete security approach.

To make sure the application remained functional and secure, I tested the program after refactoring the code to confirm that all features worked as expected. I also ran a second dependency-check scan to verify that vulnerabilities were resolved and that no new issues were introduced. This step was important to ensure that security improvements did not negatively impact the application’s performance or functionality.

Throughout this project, I used several useful resources and tools, including Maven, OWASP Dependency-Check, and Java security libraries such as MessageDigest for hashing. I also applied secure coding practices like input validation, proper error handling, and avoiding hardcoded sensitive data. These tools and practices will be helpful in future assignments and real-world development tasks.

For future employers, I would showcase this project as an example of my ability to perform a vulnerability assessment and implement secure coding practices. This artifact demonstrates my understanding of software security, my ability to use industry tools, and my experience in improving the security of an application while maintaining its functionality.
